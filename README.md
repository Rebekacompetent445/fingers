## 📖 What fingers is

`fingers` is a draft specification for the `fingers://` protocol. It is meant to describe how a simple client and server can talk over a small network with clear rules and plain text messages.

This project fits well with small net tools and older network ideas like:

- finger
- gopher
- gemini protocol
- RFC 1288
- webfinger

It also sits in the space of small web and smolnet projects, where the goal is to keep things light, direct, and easy to read.

## ✨ What you can find in this repository

This repository is not a full app. It is a specification project. That means it gives you a written guide for how the protocol should work.

You can expect content like:

- protocol goals
- request and response format
- address style for `fingers://`
- text rules for clients and servers
- examples for simple use
- notes on compatibility with small network tools

The writing is aimed at people who want a simple, text-first protocol with clear behavior.
